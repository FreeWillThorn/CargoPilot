## Summary

Add the core Order Assistant data contract so later issues can store Assistant Runs, suggestions, Review Requests, Change Drafts, source references, model usage, statuses, and Chinese UI labels consistently.

## Requirements

- Add persistent structures for Assistant Run, Assistant Suggestion, Review Request, Change Draft, Source Reference, and model usage using the repo's existing persistence style.
- Keep every record tied to one selected Import Order.
- Support suggestion levels `suggestion`, `review-needed`, and `blocking-risk` with Chinese labels `建议`, `需核查`, and `阻塞风险`.
- Support Review Request statuses `pending_review`, `approved_for_draft`, `ignored`, and `needs_followup` with Chinese labels.
- Support Change Draft statuses `draft`, `confirmed`, `rejected`, and `failed` with Chinese labels.
- Define the shared agent response envelope: `suggestions[]`, `drafts[]`, `reviewNeededFields[]`, and `usage`.
- Define common fields for suggestion target, source references, draft type, proposed values, confidence, prompt version, and model usage.
- Define enough run metadata to enforce Agent Isolation Standard: selected Import Order, current user role, task template, source references, allowed tool types, prompt version, agent name, and agent output type.
- Ensure the backend can validate agent JSON before any Assistant Suggestion, Review Request, or Change Draft is persisted.

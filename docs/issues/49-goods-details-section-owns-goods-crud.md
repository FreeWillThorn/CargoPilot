## Agent Brief

**Category:** bug

**Summary:** Convert 货物跟踪 into 货物详情 and consolidate Goods Line CRUD/status editing there.

**Current behavior:**

货物跟踪 is primarily a logistics tracking table. It still shows an `异常` column. Goods Line create/import/edit/delete is split with 订单项目, while status editing appears inline in both places.

**Desired behavior:**

The section should be labeled `货物详情`. It should be the single place for Goods Line detail management under a selected order:

- selected Import Order selector
- Goods Line list for that order
- Goods Line create/manual add
- Goods Line Excel upload
- Goods Line edit
- Goods Line delete
- Goods Logistics Status inline edit
- Supplier/product/quantity/price/package summary columns appropriate for Goods Line detail

Remove the `异常` column from this section. Arrival exceptions remain handled in 仓库盘点; cross-order exception triage should not dominate the normal 货物详情 view.

Successful create/edit/delete/import/status actions should stay in 货物详情 with the same selected order. Admin can edit all Goods Line fields; Warehouse User can update permitted logistics/warehouse-facing fields only if existing permissions allow it.

**Key interfaces:**

- Tracking/Goods Line page rendering and route labels.
- Goods Line create/edit/delete handlers.
- Goods Line Excel upload handler from Issue 44.
- Goods Logistics Status inline select.
- Active navigation route mapping for `/goods-lines/...` should move from 订单详情/old 订单项目 to 货物详情.

**Acceptance criteria:**

- [ ] Left nav/page title show `货物详情`, not `货物跟踪`.
- [ ] The table no longer has an `异常` column.
- [ ] Goods Line create/manual add and Excel upload are available from 货物详情.
- [ ] Goods Line edit/delete actions are available from 货物详情.
- [ ] Goods Logistics Status can be changed inline from the display state.
- [ ] `/goods-lines/...` edit/back links return to 货物详情 with the selected order.
- [ ] 订单详情 no longer owns Goods Line CRUD after this migration.
- [ ] Tests cover the renamed page, absence of `异常` column, create/import/edit/delete access, and status update.

**Out of scope:**

- Warehouse receiving exception workflow.
- New logistics statuses.
- Container loading visualization.

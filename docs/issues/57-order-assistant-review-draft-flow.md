## Summary

Let Admin Users process Review Requests and confirm or reject Change Drafts without allowing AI to apply changes directly.

## Requirements

- Admin Users can mark Review Requests as ignored, needs follow-up, or approved for draft creation.
- Approved Review Requests can create Change Drafts.
- Change Draft confirmation UI shows original value, AI suggested value, and administrator final value.
- Admin Users may edit the final value before confirming.
- Confirmed drafts update only the intended selected-order data.
- Rejected and failed drafts keep audit history.
- AI cannot change Order Status, warehouse receiving results, or loading records.

## Testing

- Test each Review Request status action.
- Test draft creation only after approval.
- Test confirmed drafts update the intended target.
- Test rejected/failed drafts do not update target data.
- Test protected targets cannot be changed by drafts.

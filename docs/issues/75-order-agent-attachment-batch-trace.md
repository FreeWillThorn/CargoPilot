## Summary

Let a conversation accept one input batch of Excel/PDF/TXT files and pasted text, locally parse source summaries, and show each parse step in Agent Processing Trace without generating business drafts.

## Requirements

- Accept multiple files and pasted text in one Order Agent conversation input batch.
- Support `.xlsx`, `.xls`, `.pdf`, `.txt`, and pasted text.
- Reuse existing local parsing where practical, including PDF text extraction and PDF OCR.
- Record one visible Agent Processing Trace step per meaningful action: file read, parse success, parse failure, row/page count when known, and source summary creation.
- Do not generate user-facing data-entry applications or risk findings from local parsing alone.
- If one file fails, keep the failure visible while preserving summaries from successful files.

## Acceptance Criteria

- [x] Uploading multiple supported files creates source summaries tied to the retained conversation.
- [x] Each file has a visible trace entry with success or failure.
- [x] PDF OCR failure is visible and does not look like success.
- [x] Pasted text is represented as a source summary.
- [x] No Order Creation Draft, Order Update Draft, Goods Line Draft, or risk suggestion is created by this slice.
- [x] Tests cover mixed successful and failed file parsing.

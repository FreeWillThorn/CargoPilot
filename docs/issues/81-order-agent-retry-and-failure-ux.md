## Summary

Make failed Order Agent steps clear, retryable, and safe without adding automatic retry or model switching.

## Requirements

- Show failure state at the failed Agent Processing Trace step.
- Failure must not look like success or create partial user-facing conclusions.
- Provide a manual retry action for the failed step.
- Retry reuses the same conversation and input batch.
- Do not add automatic retry, fallback model behavior, or multi-model switching.
- Preserve uploaded source summaries and user messages across retry.

## Acceptance Criteria

- [ ] Task Understanding failure shows a clear failed step and retry action.
- [ ] Data Entry failure shows a clear failed step and retry action.
- [ ] Risk Agent failure shows a clear failed step and retry action.
- [ ] Retrying reuses the same conversation and input batch.
- [ ] Failed runs create no fallback drafts or risks.
- [ ] Tests cover retry for at least one model step and one parsing step.

## Agent Brief

**Category:** bug

**Summary:** Let 订单项目 add Goods Lines either manually or by uploading the provided Chinese Excel template.

**Current behavior:**

In an Import Order's 货物明细 panel, the add action only opens a manual Goods Line form. Excel import exists elsewhere and uses an old English template/path-based workflow, so it does not match the order-centered UI or the user-provided Goods Line template.

**Desired behavior:**

In 订单项目, the 货物明细 action should open a drawer or simple action choice with:

- 手动添加货物项
- 上传 Excel 货物清单

Manual add should keep the current behavior. Excel upload should use a browser file input (`type=file`) and the current selected Import Order, so the template does not need an order number column.

The provided template columns should map as follows:

- `产品名称` -> Goods Line Chinese product name
- `数量（非包裹数）` -> quantity
- `实际付款` -> purchase amount source; if numeric, derive purchase unit price by dividing by quantity; if blank or `-`, leave price empty
- `链接` -> product URL
- `厂家名称` -> Supplier name; create the Supplier if it does not exist

Use sensible defaults for missing fields: unit `pcs`, empty SKU/model, empty customs English name, and current Import Order from the page context. Import should report row-level errors without creating a broken row when required values are missing or invalid.

Uploaded workbooks should be accepted through the existing multipart parser and saved under the local uploads/imports area before parsing, or parsed from a safe temporary local copy if the existing upload helper already supports that cleanly.

**Key interfaces:**

- A Goods Line import function should accept an Import Order id and an uploaded workbook path/bytes.
- The web handler should support multipart form upload from 订单项目.
- The result should report created count and row errors back on the 订单项目 page or the same selected order context.

**Acceptance criteria:**

- [ ] 货物明细 add action presents both manual add and Excel upload options.
- [ ] The upload form uses a real local file upload control, not a path-only input.
- [ ] Uploading a workbook with the provided Chinese headers creates Goods Lines under the currently selected Import Order.
- [ ] Supplier names from `厂家名称` create or reuse Supplier records.
- [ ] Numeric `实际付款` is converted into unit purchase price when quantity is valid.
- [ ] Blank or `-` `实际付款` does not block importing the row.
- [ ] Row-level import errors are shown without leaving the selected Import Order context.
- [ ] Tests cover a successful Chinese-template upload and one invalid-row error.

**Out of scope:**

- Smart recognition of arbitrary Excel formats.
- Supplier package/logistics import.
- Customs English name generation.
- AI-based product classification.

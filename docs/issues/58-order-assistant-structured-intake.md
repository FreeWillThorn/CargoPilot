## Agent Brief

**Category:** enhancement

**Summary:** Teach Structured Intake Agent to turn a complete Chinese goods-list workbook into selected-order Goods Line drafts and review-needed fields.

**Current behavior:**

Order Assistant Structured Intake is specified at a high level, but Issue 58 does not yet define a concrete Excel acceptance case. The product already has direct fixed-template Goods Line upload work covered elsewhere, but the AI assistant workflow still needs a draft-based path for uploaded Excel sources.

**Desired behavior:**

When an Admin User uploads a complete Chinese goods-list Excel source through the selected Import Order's Order Assistant drawer, Structured Intake Agent should extract proposed Goods Line Change Drafts tied to that selected Import Order. The drafts are not applied to `货物详情` until the Admin User confirms them.

The workbook shape represented by `货物清单导入-已补全.xlsx` should be supported as an MVP sample. The implementation should either commit a stable test fixture derived from that workbook or generate an equivalent workbook in tests, so tests do not depend on an untracked local file.

Column mapping:

- `产品名称` -> Goods Line product name / Chinese product name.
- `数量（非包裹数）` -> quantity; if the cell includes a unit such as `60米`, preserve the source value and mark it for review rather than guessing.
- `箱数量` -> carton count.
- `实际付款` -> purchase amount source or finance draft evidence; numeric values may derive purchase unit price only when quantity is safely numeric. `-` leaves the price empty or review-needed.
- `链接` -> product URL.
- `厂家名称` -> Supplier name reference for matching or a suggestion to add/select in `基础资料`; do not create Supplier master-data drafts.
- `外箱尺寸(cm)` -> carton length/width/height when it can be safely parsed from `L*W*H`.
- `单箱毛重(kg)` -> carton gross weight.
- `CBM` -> total CBM source value.
- `总毛重(kg)` -> total gross weight source value.
- `麦头` -> Shipping Mark.
- `国内物流单号` -> Domestic Tracking Number.
- `货物物流状态` -> review-needed logistics status source; do not create warehouse receiving results from this field.

Low-confidence or missing cells should become Review-Needed Fields attached to the affected proposed Goods Line draft. Rows with enough identifying information, such as product name and quantity, should still produce a draft even when supplier/package/logistics values need review.

**Key interfaces:**

- Structured Intake Agent response envelope should use `drafts[]` for proposed Goods Line drafts and finance/document-data drafts.
- `reviewNeededFields[]` should capture low-confidence values, missing optional-but-important values, and mixed value/unit cells.
- Source References should point back to the uploaded workbook, row number, and column header.
- Structured Intake Agent must stay inside its Agent Contract: extraction and draft candidates only, no direct imports, no master-data creation, no reading unrelated orders.
- Change Draft confirmation remains the only path that writes to system data.

**Acceptance criteria:**

- [ ] A workbook with the sample Chinese headers produces multiple Goods Line Change Drafts for the selected Import Order.
- [ ] Drafts include product name, quantity, carton count, product URL, supplier name reference, parsed package dimensions, carton gross weight, CBM, total gross weight, Shipping Mark, Domestic Tracking Number, and logistics status source when present.
- [ ] The sample's `-` payment values do not block draft creation.
- [ ] Mixed quantity/unit values such as `60米` become Review-Needed Fields instead of guessed numeric quantities.
- [ ] Missing supplier/package/logistics values on an otherwise identifiable row become Review-Needed Fields, not a broken applied row.
- [ ] Supplier names from `厂家名称` do not create Supplier master-data drafts.
- [ ] Source References include workbook row and column evidence for proposed fields.
- [ ] Structured Intake output uses the shared structured JSON envelope and contains no free-form business action instructions.
- [ ] Confirming a Change Draft is still required before any Goods Line appears or changes in `货物详情`.

**Out of scope:**

- Direct Goods Line import that bypasses Change Draft review; that belongs to the fixed Excel upload workflow.
- Creating Supplier, Customer/Consignee, Warehouse, or Company/System master data from AI intake.
- Creating warehouse receiving results from `货物物流状态`.
- Smart recognition of arbitrary unrelated Excel layouts.

## Agent Brief

**Category:** bug

**Summary:** Rework 成本利润 cost/charge management into current-section CRUD drawers with compact icon actions.

**Current behavior:**

成本利润 has a selected Import Order and summary, but 新增成本/收费 is shown as a flat form on the page. Existing cost and charge rows can be reviewed, but they do not expose compact per-row edit/delete actions. This breaks the UI rule used elsewhere: object actions should be launched from the current section with drawer/modal actions, then return to the same selected context.

**Desired behavior:**

成本利润 should keep the selected Import Order context. Manual finance-line actions should be available as compact section actions:

- 新增成本
- 新增客户收费
- 编辑成本/收费行
- 删除成本/收费行

Create/edit actions should open as right-side drawers. Delete should use the simplest existing confirmation pattern. Successful create/edit/delete should redirect back to 成本利润 with the same selected Import Order. Only Admin Users may create, edit, or delete finance lines.

The UI should keep Chinese-first labels. Internal enum values such as `line_kind`, `cost_type`, and `charge_type` should not appear as visible field labels, though accepted finance terms may remain as option values until a later label-polish issue.

**Key interfaces:**

- Finance line creation should keep using the existing finance-line model: Import Order, optional Goods Line, cost/charge kind, line type, amount, currency, exchange rate, and notes.
- Add or reuse finance-line update/delete operations so rows can be edited or removed safely.
- The browser UI should reuse existing drawer/modal and icon-action patterns rather than introducing a component framework.

**Acceptance criteria:**

- [ ] 成本利润 no longer shows the manual 新增成本/收费 form as a permanently expanded page block.
- [ ] 新增成本 and 新增客户收费 are launched from compact action buttons and open drawer forms.
- [ ] Cost and charge tables show compact edit/delete actions for each row.
- [ ] Admin can create, edit, and delete a finance line and remain on the same selected Import Order.
- [ ] Warehouse User cannot create, edit, or delete finance lines.
- [ ] Tests cover at least one create/edit/delete flow and the role restriction.

**Out of scope:**

- Supplier payment tracking.
- Customer payment tracking.
- Accounting ledger behavior.
- Excel cost import; that is covered by Issue 45.

## Agent Brief

**Category:** bug

**Summary:** Add Excel upload for importing cost lines in 成本利润 using the provided cost template.

**Current behavior:**

成本利润 can add finance lines manually, but there is no Excel upload for the provided cost template. The existing Excel import area is path-based and targets Goods Line purchase/package data, not finance cost lines.

**Desired behavior:**

Within 成本利润 for the selected Import Order, 新增成本 should support:

- 手动添加
- 上传 Excel 成本

The upload form should use a real browser file input (`type=file`) and stay in the selected Import Order context after submit. Uploaded cost rows should create order-level finance lines with `line_kind=cost`.

The provided template columns should map as follows:

- `中文项目 (Item)` -> user-facing cost item label; include in notes
- `English Description` -> description; include in notes and use for simple line type mapping when obvious
- `Amount` -> amount
- `Currency` -> currency
- `说明备注 (Remarks)` -> notes

Minimal line type mapping is enough:

- English description containing `Ocean Freight` -> `sea_freight`
- otherwise -> `other`

Use exchange rate `1` by default. Invalid amount rows should be reported as row-level errors and skipped.

**Key interfaces:**

- A finance cost import function should accept an Import Order id and uploaded workbook path/bytes.
- The web handler should support multipart upload from 成本利润.
- The import should create finance lines using the existing finance-line model.

**Acceptance criteria:**

- [ ] 新增成本 offers manual add and Excel upload options.
- [ ] The upload form uses a real local file upload control, not a path-only input.
- [ ] Uploading the provided cost template creates cost finance lines for the selected Import Order.
- [ ] `Ocean Freight` rows are imported as `sea_freight`; other rows are imported as `other`.
- [ ] Amount and currency are preserved.
- [ ] Row-level errors are shown for invalid amounts without blocking valid rows.
- [ ] Tests cover a successful Chinese/mixed-header cost upload and one invalid-row error.

**Out of scope:**

- Importing customer charge lines from Excel.
- Exchange-rate lookup.
- Arbitrary cost category inference beyond the minimal mapping above.
- Payment status or accounting ledger behavior.

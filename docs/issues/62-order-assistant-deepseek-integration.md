## Summary

Integrate DeepSeek for validated structured JSON model calls while keeping demo mode and administrator confirmation safeguards.

## Requirements

- Configure DeepSeek with `DEEPSEEK_API_KEY` and `DEEPSEEK_MODEL`.
- Use demo mode when no API key is configured.
- Validate structured JSON output before creating suggestions, Review Requests, or Change Drafts.
- Store model name, run time, input token count, output token count, selected Import Order, and prompt version.
- Require administrator confirmation before sending real customer, supplier, pricing, or document data to the external model API.
- Do not store full model reasoning text.
- Failed model calls mark the relevant Assistant Run or Specialist Agent as failed without partial suggestions.
- DeepSeek prompts and schemas must preserve agent isolation: one agent contract per model call, selected-order context only, no free-form business actions, and validated JSON output only.

## Testing

- Test demo mode without API key.
- Test invalid JSON is rejected.
- Test model usage logging.
- Test external-send confirmation is required for real data.
- Test failed calls do not create partial suggestions.
- Test a model response that attempts a forbidden action is rejected before persistence.

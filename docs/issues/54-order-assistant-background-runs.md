## Summary

Add the Assistant Run lifecycle so an AI action can start in the background, show queued/running/succeeded/failed state, retry failed runs, and record prompt/model usage.

## Requirements

- Support Assistant Run statuses `queued`, `running`, `succeeded`, and `failed`.
- Starting a run must not change existing order data.
- Failed runs can be retried.
- Store prompt version on each run.
- Store model name, run time, input token count, output token count, and selected Import Order.
- Add demo mode behavior when no `DEEPSEEK_API_KEY` is configured.

## Testing

- Test run status transitions.
- Test retry from failed status.
- Test model usage recording.
- Test demo mode without external model access.

## Agent Brief

**Category:** bug

**Summary:** Convert 订单项目 into 订单详情 focused on order details, order CRUD, and embedded 收货客户 CRUD.

**Current behavior:**

The order section is labeled 订单项目 and mixes order summary with Goods Line detail. It allows adding orders, but full order edit/delete CRUD is not available in the same drawer/icon-action style used elsewhere. 收货客户 CRUD is separate under 管理/设置.

**Desired behavior:**

The section should be labeled `订单详情`. It should focus on:

- all orders and order status
- selected order detail
- order create/edit/delete actions
- 收货客户 create/edit/delete actions needed while creating/editing orders

Goods Line detail/table/create/import/edit/delete should be removed from this section once 货物详情 owns it. The section may still show order-level aggregate counts such as total goods count, cartons, CBM, and gross weight, but not the full Goods Line CRUD table.

Order create/edit should use the same drawer pattern as current object actions. Delete/cancel should use the simplest existing confirmation pattern. After create/edit/delete, remain on 订单详情 and preserve the selected order context where possible.

**Key interfaces:**

- Order list and selected-order detail rendering.
- Import Order create/update/delete operations.
- Consignee create/update/delete operations or current master-data helpers.
- Role permissions: Admin can edit; Warehouse User can view only.

**Acceptance criteria:**

- [ ] Left nav/page title show `订单详情`, not `订单项目`.
- [ ] 订单详情 no longer renders the full Goods Line/货物明细 CRUD table.
- [ ] Admin can create, edit, and delete/cancel an order from 订单详情 using compact actions and drawers/modals.
- [ ] Admin can create, edit, and delete 收货客户 from within 订单详情.
- [ ] Warehouse User can view order details but cannot create/edit/delete orders or consignees.
- [ ] Order status remains editable inline for Admin Users.
- [ ] Tests cover order edit, order delete/cancel, embedded consignee create, and Warehouse restriction.

**Out of scope:**

- Moving Goods Line CRUD; that belongs to Issue 49.
- Customer login portal.
- Accounting/payment behavior.

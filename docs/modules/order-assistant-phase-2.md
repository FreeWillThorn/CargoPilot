# Order Assistant Phase 2

## Trigger

Start this phase after MVP usage shows repeated low-risk Change Drafts that Admin Users usually approve without edits.

## Add Next

- Batch confirmation for low-risk Change Drafts.
- Configurable agent routing and saved run templates after fixed MVP routing proves useful.
- Saved assistant run templates per workflow section.
- Better arbitrary Excel/PDF/chat extraction after real sample files reveal stable patterns.
- Admin-configurable compliance hint rules for product names and categories.
- Configurable profit-risk thresholds and required fee categories.
- Optional assistant reminders on Dashboard for repeated unresolved Review Requests.
- Optional dedicated assistant workspace if the embedded Order Assistant entry becomes too crowded.
- AI cost dashboard if model usage becomes meaningful enough to manage.
- Prompt management UI only if non-developers need to tune prompts safely.
- Configurable routing UI only after task templates and source rules are stable.

## Keep

- Import Order remains the required starting context.
- Official documents still require Admin User confirmation.
- Blocking-risk suggestions remain visible as administrator decisions, not AI decisions.

## Do Not Add Yet

- Autonomous order changes without Admin User confirmation.
- Supplier or customer-facing assistant actions.
- External email, SMS, or WeChat notifications.
- Live carrier, customs, or government certificate integrations.

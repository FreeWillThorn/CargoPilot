# Shadcn 风格 Dashboard 美化留痕

## 背景

本次 UI 美化按用户选定的 `shadcn Dashboard Blocks` 风格执行，目标是把 Dashboard 从普通概览页调整为货代调度台：先暴露风险，再推进订单。

## 修改范围

- `cargopilot/web.py`
  - 全局壳层改为更接近 shadcn 的白底、细边框、低饱和配色。
  - 顶部 Header 增加面包屑、全局搜索和用户操作区。
  - Dashboard 改为 `今日调度台`：指标卡、订单处理队列、紧急队列、AI 建议。
  - 新增纯展示 helper：优先级、下一步、风险徽标、AI 建议文案。
- `tests/test_web.py`
  - 更新 Dashboard 断言以覆盖新文案和关键链接。

## 不变项

- 没有新增前端依赖。
- 没有改数据库结构。
- 没有改订单、货物、仓库、单证、财务的写入逻辑。
- 表单 `action`、主要链接、状态筛选和现有路由保持可用。

## 设计约束

- 中文优先。
- 表格仍是主工作面，不改成卡片列表。
- 风险颜色只用于徽标和数字，不大面积铺色。
- 长表格继续使用滚动容器，避免真实数据撑坏页面。

## 后续可选

- 将同一套视觉 token 扩展到 订单详情、货物详情、仓库盘点。
- 将 Dashboard 的目的港、聚焦点、预计装柜日筛选接入真实后端过滤。
- 以后若改为 React，可迁移到真实 shadcn/ui 组件；当前 Python 原生 HTML 只借鉴视觉语言。
